# Workflows

## Purpose
Jupyter notebooks containing experiment designs and workflows. Each notebook represents a self-driving lab experiment that orchestrates hardware through the backend library functions.

## Structure
Each workflow notebook should:
1. Import necessary device libraries from `backend.src.devices`
2. Import MQTT controllers and utilities
3. Define high-level experiment steps using device abstractions
4. Include documentation and results
